LOD Visualization, Analytics and DiscovEry in Real-time.

{{appName}} a new hybrid system which combines LOD Visualisation, Analytics and DiscovEry to aid in answering questions as "What kind of datasets are in the LOD cloud?", "In what way(s) are these datasets connected?". Moreover, {{appName}} indexes and count valid and invalid links among datasets, creating a regular LOD diagram and a "dark LOD diagram". {{appName}} is also equipped with a search engine, making fast to find subjects, properties and objects among billions of triples. Our diagram page shows a demo using Data Driven Documents JS Library (D3).

A brief explanation...

{{appName}} uses Bloom filters (BF) to compare, index and count links between RDF distributions in the streaming process. How do we do that?

  • 1. {{appName}} parses your dataset description file that might be in different formats such as VoID, DCAT and DataID. Then, we stream your RDF data in order to extract resources and compare with different Bloom filters which contains index for other distributions.
  • 2. We retrieve RDF data about links (void:linkset format) of your dataset and we create new Bloom filters representing your dataset. Later these Bloom filters will be used to compare your datasets with previously loaded datasets.
  • 3. A LOD-Diagram is provided showing the links that we discovered of your datasets.
  • 4. We fetch statistical data using Jaccard Index and top N approaches.
  • 5. You can use a REST API to retrieve links indegree and outdegree of your RDF data.

We provide an online demo which uses our API. Click in "Quick Start" and check how easy is to use add datasets to {{appName}}. After that, go to our diagram page to visualize the linked resources.
